Mobile-first + auto-fit + minmax — SEMPRE
Nunca usar grid-template-columns: 1fr 1fr fixo. Sempre repeat(auto-fit, minmax(min(100%, Xpx), 1fr)). O min(100%, Xpx) garante que nenhum card ultrapasse o container em mobile.
grid-2col — 2 Colunas (min 340px)
grid-template-columns: repeat(auto-fit, minmax(min(100%, 340px), 1fr));
Card A — estrutura simétrica
Colapsa para 1 col em mobile (< 340px disponível). Padding 20px. Mesma estrutura interna dos cards do lado.
Card B — estrutura simétrica
Ambos os cards devem ter o mesmo padding, altura aproximada e estrutura interna. Simetria visual é obrigatória.

Regras de Simetria
✓ Correto
· Todos os cards no grid com mesmo padding
· Todos os cards com mesma variant (default ou accent)
· gap horizontal = gap vertical
· auto-fit + minmax com min(100%, X)
· border-radius consistente entre cards irmãos
✗ Errado
· Misturar card default e card accent-left no mesmo grid
· padding: 20px em um, padding: 14px em outro
· gap: 16px 8px (assimétrico)
· grid-template-columns: 1fr 1fr (sem auto-fit)
· Card com 3 elementos internos ao lado de card com 1
8 Padrões Sistêmicos — Lawfare Timeline
Estes padrões recorrem em todas as operações documentadas. Cada artefato deve identificar quais padrões estão presentes e marcá-los com a cor correta.
P1
Anulação via Defecto Processual
HC, nulidades e recursos processuais como instrumento de blindagem. Investigados liberados antes do fim do processo.
CRÍTICO Lava Jato · Hydra · Compliance Zero
P2
Investigadores/Delatores se Tornam Alvos
Whistleblowers executados, investigadores processados. Inversão do vetor de perseguição. O informante vira o perseguido.
CRÍTICO Hydra (Gritzbach) · Satiagraha · Castelo de Areia
P3
Captura Judicial Emergencial — STF como Terminal
Casos que chegam ao STF param antes dos beneficiários finais. Sigilo judicial como mecanismo de proteção.
ESPECIAL Compliance Zero · Hydra · Lava Jato
P4
Weaponização da Mídia / Desinformação
Campanhas de fake news para reverter regulações, desgastar investigadores ou criar narrativas alternativas.
ALTO Hydra (e-Financeira) · 8 de Janeiro
P5
Uso de Fundos Públicos como Vetores
INSS, FGC, BRB, Petrobras como artérias do esquema. O cidadão financia a própria espoliação via tributos e contribuições.
ALTO Sem Desconto · Compliance Zero · Lava Jato
P6
Uso Estratégico da Prescrição
Investigações deliberadamente retardadas até que crimes prescrevam. O tempo como arma jurídica contra a accountability.
MÉDIO Castelo de Areia · Satiagraha · Poço de Lobato
P7
Captura Transgeracional — Previdência Social
Fraudes em benefícios previdenciários e consignados afetando aposentados de forma sistemática e continuada.
ALTO Sem Desconto (INSS) · Compliance Zero
P8 — NOVO
Captura do Sistema de Pagamentos Digital
Infiltração de crime organizado em fintechs reguladas como 'instituições de pagamento'. Crime se antecipa à regulação em 5+ anos.
EMERGENTE Hydra (2GO/InvBank) · Carbono Oculto (BK Bank)
Checklist para AIs — Criação de Artefatos
Antes de Começar
1. Identificar o tipo de artefato (dossie, dashboard, mapa mental, timeline, jurídico, popular)
2. Copiar o :root CSS template — nunca modificar as variáveis base
3. Usar o combined_import das Google Fonts (Syne + IBM Plex Mono)
4. Definir a cor accent da operação (red = crimes graves, amber = investigação, blue = judicial)
Grids e Layout
5. Todos os grids: auto-fit + minmax(min(100%, Xpx), 1fr) — nunca colunas fixas
6. Começar pelo mobile (base CSS) → adicionar @media (min-width: 768px) para desktop
7. Padding lateral: 20px mobile → 40px desktop
8. Verificar simetria: todos os cards num grid com mesmo padding e estrutura
Conteúdo e Semântica
9. Usar badges com a cor correta do severity_map para cada status
10. Identificar e marcar os padrões P1-P8 presentes na operação
11. Separar claramente fatos documentados de hipóteses analíticas
12. Incluir @media print e footer CC0 obrigatório
Anti-Patterns — O Que Nunca Fazer
Visual
✗ Fundo branco ou claro
✗ Inter, Roboto, Arial
✗ Gradientes decorativos
✗ Sombras pesadas
✗ border-radius > 8px em cards
✗ Cores sem semântica
✗ 4+ cols fixas em mobile
Conteúdo
✗ Omitir CC0 footer
✗ Afirmação sem fonte
✗ Hipótese como fato
✗ Omitir contestação da defesa
✗ Identificar beneficiários sem prova
✗ Uso de ! para enforcement
✗ Linguagem de conclusão prematura
Código
✗ localStorage / sessionStorage
✗ CDN externo para CSS crítico
✗ Form tags (usar div + onclick)
✗ !important para layout
✗ Omitir meta viewport
✗ grid: 1fr 1fr fixo
✗ Inline styles em loop
Árvoree de Decisão Rápida
COR: É status? → severity_map. É conexão SVG? → solid/dashed/dotted. É decorativo? → NÃO usar cor.
GRID: 1 item → 1col. 2-3 comparáveis → grid_2col. 4-6 → grid_3col. Métricas → grid_4col. Sempre auto-fit.
BADGE: Crítico/ativo/preso → red. Pendente/risco → amber. Judicial/info → blue. Confirmado → green. STF/geo → purple.
CARD: Texto/análise → default. Destaque/padrão → accent-left. Alerta → alert variant. Métrica → stat-item.
FONTE: Título/número → Syne. Todo resto → IBM Plex Mono. Nunca Inter/Roboto/Arial.
